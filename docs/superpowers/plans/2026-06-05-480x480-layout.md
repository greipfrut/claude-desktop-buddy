# 480×480 Native Layout (Phase 2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the buddy UI from a 240×320 strip centered on the panel (black borders) to a full 480×480 native layout — Big Buddy home, a roomy scrollable approval screen, and all other screens re-laid-out for the square — with the buddy scaled up to fill it.

**Architecture:** The canvas grows from 240×320@(120,80) to 480×480@(0,0); touch maps 1:1; every screen's draw code reflows against `W=H=480`. The buddy's existing `_scale` mechanism and `character.cpp`'s existing integer GIF-scale factors are bumped to fill the larger center. A new vertical drag gesture (added to `pollGesture()`) scrolls the approval hint.

**Tech Stack:** PlatformIO + Arduino-ESP32 (pioarduino), Arduino_GFX (`Arduino_Canvas`), existing buddy/GIF renderers.

**This is Phase 2 of the spec** `docs/superpowers/specs/2026-06-05-480x480-layout-and-battery-design.md`. Phase 1 (battery) is committed (`ed821e0`). The brainstorm mockups are in `.superpowers/brainstorm/271-1780635170/content/` (`home-bigbuddy.html`, `approval.html`).

**Verification model:** Embedded firmware, no unit tests. Each task is verified by (a) `pio run` compiling clean and (b) on-device observation. **Screen pixel coordinates are tuned on hardware** — each screen task lists concrete starting coordinates from the approved mockups, then a tune step (this matches how all prior UI work in this repo was done). Build/flash (PowerShell, COM4):

```powershell
& "C:\Users\tsphan-ahc\.platformio\penv\Scripts\pio.exe" run -e waveshare-s3-touch-lcd-4b
& "C:\Users\tsphan-ahc\.platformio\penv\Scripts\pio.exe" run -e waveshare-s3-touch-lcd-4b -t upload --upload-port COM4
```

**Execution order matters:** Task 1 (geometry) first — afterwards the *old* screens render mis-placed in the top-left until each is reflowed; that's expected and each subsequent task fixes one screen. Verify touch still lands correctly after Task 1 before proceeding.

**CDC note:** `platformio.ini` carries the diagnostic `ARDUINO_USB_CDC_ON_BOOT=0` (app `Serial` → COM4). Keep it at `0` through Tasks 1–6 (you'll want serial during the reflow), then flip to `1` in Task 7 (the final production setting).

```cpp
static void drawApproval() {
  const Palette& p = characterPalette();
  spr.fillScreen(p.bg);

  uint32_t waited = (millis() - promptArrivedMs) / 1000;
  spr.setTextSize(2);
  spr.setTextColor(waited >= 10 ? HOT : p.textDim, p.bg);
  spr.setCursor(12, 12);
  spr.printf("approve? %lus", (unsigned long)waited);

  // Full tool name (size 3, wraps via cursor; clip to one line at this size).
  spr.setTextSize(3);
  spr.setTextColor(p.text, p.bg);
  spr.setCursor(12, 44);
  spr.print(tama.promptTool);

  // Scrollable hint area: y 84..360, full text wrapped at ~38 cols (size 2).
  const int HINT_Y = 84, HINT_H = 276, COL_PX = 12, LINE_H = 22;
  const int wrapCols = (W - 28) / COL_PX;   // chars per line at size 2
  spr.setTextSize(2);
  spr.setTextColor(p.body, p.bg);
  // Word-wrap promptHint into lines, then draw from hintScroll offset.
  int n = strlen(tama.promptHint);
  int totalLines = 0;
  // First pass: count wrapped lines (simple hard wrap at wrapCols).
  for (int i = 0; i < n; i += wrapCols) totalLines++;
  if (totalLines == 0) totalLines = 1;
  int maxScroll = totalLines * LINE_H - HINT_H;
  if (maxScroll < 0) maxScroll = 0;
  if (hintScroll > maxScroll) hintScroll = maxScroll;
  if (hintScroll < 0) hintScroll = 0;
  int yline = HINT_Y - hintScroll;
  char line[64];
  for (int i = 0; i < n; i += wrapCols) {
    int len = (n - i < wrapCols) ? (n - i) : wrapCols;
    if (len > 63) len = 63;
    memcpy(line, tama.promptHint + i, len); line[len] = 0;
    if (yline + LINE_H > HINT_Y && yline < HINT_Y + HINT_H) {
      spr.setCursor(14, yline); spr.print(line);
    }
    yline += LINE_H;
  }
  // Scrollbar indicator when content overflows.
  if (maxScroll > 0) {
    int trackH = HINT_H, barH = trackH * HINT_H / (totalLines * LINE_H);
    int barY = HINT_Y + (trackH - barH) * hintScroll / maxScroll;
    spr.fillRect(W - 6, HINT_Y, 4, trackH, p.bg);
    spr.fillRect(W - 6, barY, 4, barH, p.textDim);
  }

  // Big OK / NO buttons.
  int btnY = H - 84, btnH = 76;
  if (responseSent) {
    spr.setTextColor(p.textDim, p.bg); spr.setTextSize(3);
    drawCenteredString(canvas, "sent", W / 2, btnY + btnH / 2);
  } else {
    spr.fillRoundRect(8,         btnY, W/2 - 12, btnH, 12, GREEN);
    spr.fillRoundRect(W/2 + 4,   btnY, W/2 - 12, btnH, 12, HOT);
    spr.setTextSize(4);
    spr.setTextColor(0x0000, GREEN); drawCenteredString(canvas, "OK", W/4,     btnY + btnH/2);
    spr.setTextColor(0x0000, HOT);   drawCenteredString(canvas, "NO", 3*W/4,   btnY + btnH/2);
  }
}
```
